## Overview

This document defines the deal flow process for NOTALONE Growth Fund, from initial sourcing through investment and Growth Assets activation.

**Fund Focus:** Post-TGE OTC investments (80%) in projects needing growth support, with selective pre-TGE positions (20%) in companies we'll support post-launch.

---

## Stage 1: Sourcing

### 1.1 Deal Sources

| Source | Description | Expected Quality |
|--------|-------------|------------------|
| **Services Pipeline** | Projects already engaged with Notalone services | Highest (known entity) |
| **ICP Hubs Network** | 24 partner organizations across regions | High (pre-vetted) |
| **Conference Presence** | Chain Fusion Houses, Token2049, Devcon, etc. | Medium-High |
| **LP Referrals** | Introductions from limited partners | Medium-High |
| **VC Network** | 1,000+ funds/accelerators in network | Medium |
| **Inbound Applications** | Direct outreach from projects | Variable |

### 1.2 Sourcing Criteria (Quick Filter)

| Criterion | Requirement | Rationale |
|-----------|-------------|-----------|
| Token Status | Post-TGE with live token | OTC focus |
| Market Cap | $10M - $500M | Sweet spot for growth impact |
| Token Liquidity | >$100K daily volume | Entry/exit feasibility |
| Team Responsiveness | <48hr reply | Engagement indicator |

**Deliverable:** Deal log entry with source, basic metrics, initial contact

---

## Stage 2: Initial Screening

### 2.1 Screening Call (30 min)

**Objective:** Determine if project fits NOTALONE's growth-focused thesis

**Discussion Points:**
1. Current traction and growth challenges
2. Why they're seeking OTC capital specifically
3. What growth support they need (if any)
4. Team composition and decision-making
5. Timeline and urgency

### 2.2 Screening Criteria

| Category | Criteria | Pass/Fail |
|----------|----------|-----------|
| **Growth Need** | Clear gap in adoption, BD, or ecosystem | Required |
| **Team Fit** | Willing to engage with Growth Assets | Required |
| **Token Health** | No major unlock cliffs in 6 months | Required |
| **Market Position** | Differentiated product or narrative | Required |
| **Cap Table** | No predatory early investors | Required |

### 2.3 Project Classification

After screening, classify the opportunity:

| Type | Description | Next Step |
|------|-------------|-----------|
| **OTC + Growth Assets** | Needs capital + active growth support | Full DD |
| **OTC Only** | Capital need, limited growth engagement | Simplified DD |
| **Services First** | Not ready for investment, services fit | Refer to Services |
| **Pass** | Doesn't fit criteria | Decline with feedback |

**Deliverable:** Screening memo (1 page) with classification and recommendation

## Stage 3: Due Diligence

### 3.1 Investment Due Diligence

| Area | Key Questions | Sources |
|------|---------------|---------|
| **Token Economics** | Inflation schedule, unlock cliffs, treasury size | Tokenomics docs, on-chain |
| **On-Chain Metrics** | Active users, TVL, transaction volume, trends | Dune, DefiLlama, Token Terminal |
| **Competitive Position** | Market share, differentiation, alternatives | Research, competitor analysis |
| **Team Background** | Track record, prior exits, reputation | LinkedIn, references, social |
| **Cap Table** | Early investor terms, insider selling patterns | Direct disclosure, on-chain |
| **Legal Status** | Token classification, regulatory exposure | Legal review |

### 3.2 Growth Fit Due Diligence

| Area | Key Questions | Assessment |
|------|---------------|------------|
| **Growth Gaps** | What's blocking adoption? Technical? BD? Marketing? | Interview team |
| **Audience Alignment** | Which persona(s) would benefit? Builders/Investors/Strategic | Map to Growth Assets |
| **Geographic Fit** | Which regions are priorities? | Match to conference calendar |
| **Engagement Capacity** | Does team have bandwidth to participate in events? | Interview team |
| **Content Readiness** | Speaking topics, demos, materials available? | Review assets |

### 3.3 OTC Terms Analysis

| Element | Target Range | Notes |
|---------|--------------|-------|
| Discount to spot | 25-60% | Depends on liquidity, lock-up |
| Investment size | $100K - $500K | Per deal |
| Vesting period | 0-12 months | Post-TGE standard |
| Cliff | 0-3 months | If any |
| Growth Assets lock | 30-50% of tokens | If applicable |

**Deliverable:** Due diligence report with recommendation

---

## Stage 4: Investment Committee

### 4.1 IC Memo Structure

| Section | Content |
|---------|---------|
| **Executive Summary** | 2-3 sentences on opportunity |
| **Investment Thesis** | Why this project, why now |
| **Deal Terms** | Size, discount, vesting, Growth Assets structure |
| **Growth Plan** | How we'll deploy Growth Assets |
| **Risk Assessment** | Key risks and mitigations |
| **Return Scenario** | Base, bull, bear cases |
| **Recommendation** | Invest/Pass with conviction level |

### 4.2 IC Decision Framework

| Factor | Weight | Scoring |
|--------|--------|---------|
| Token fundamentals | 25% | 1-5 scale |
| Growth opportunity | 25% | 1-5 scale |
| Team quality | 20% | 1-5 scale |
| Deal terms | 15% | 1-5 scale |
| Strategic fit | 15% | 1-5 scale |

**Minimum score to proceed:** 3.5/5.0

### 4.3 IC Outcomes

| Decision | Next Step |
|----------|-----------|
| **Approve** | Proceed to term sheet |
| **Approve with conditions** | Address conditions, return to IC |
| **Request more DD** | Specific areas to investigate |
| **Decline** | Document reasons, maintain relationship |

**Deliverable:** IC decision memo with terms approval

## Stage 5: Deal Structuring

### 5.1 Term Sheet Elements

**Investment Terms:**
| Element | Specification |
|---------|---------------|
| Investment amount | $X USD |
| Token amount | Y tokens at $Z reference price |
| Discount | X% to [7-day VWAP / agreed price] |
| Immediate vest | X% (matches cash investment value) |
| Locked tokens | Y% (Growth Assets collateral) |

**Growth Assets Terms:**
| Element | Specification |
|---------|---------------|
| Lock structure | Base / Activity / Performance pools |
| Release triggers | Specific conference participation, milestones |
| Package entitlements | Which Growth Assets packages included |
| Exit provisions | Startup exit rights, Notalone obligations |

### 5.2 Legal Documentation

| Document | Purpose |
|----------|---------|
| Token Purchase Agreement | OTC investment terms |
| Growth Assets Agreement | Service delivery commitments |
| Smart Contract Specs | Token lock and release mechanics |
| Side Letter (if needed) | LP-specific terms |

**Deliverable:** Signed agreements, smart contract deployed

## Stage 6: Onboarding

### 6.1 Kickoff Call

**Participants:** Notalone deal lead, project founders, Growth Assets coordinator

**Agenda:**
1. Relationship introduction and contacts
2. Review Growth Assets entitlements
3. Map to upcoming conference calendar
4. Identify first activation opportunities
5. Content and asset requirements
6. Communication cadence

### 6.2 Onboarding Checklist

| Task | Owner | Timeline |
|------|-------|----------|
| Add to portfolio dashboard | Notalone | Day 1 |
| Create shared workspace | Notalone | Day 1 |
| Collect brand assets | Project | Week 1 |
| Speaker bios and topics | Project | Week 1 |
| Map to Q1 conferences | Together | Week 1 |
| Introduce to relevant partners | Notalone | Week 2 |

### 6.3 Growth Assets Planning

| Element | Discussion |
|---------|------------|
| Priority personas | Which of Builders/Investors/Strategic? |
| Geographic focus | Which regions first? |
| Content readiness | Speaking topics, demos, materials |
| Team availability | Who travels, who speaks? |
| Success metrics | What outcomes matter most? |

**Deliverable:** Onboarding complete, first activation scheduled

## Stage 7: Growth Assets Activation

### 7.1 Conference Activation Cycle

**Pre-Conference (4-6 weeks before):**
| Task | Owner |
|------|-------|
| Confirm attendance | Project |
| Submit to speaker roster | Notalone |
| Coordinate house event slot | Notalone |
| Prepare materials | Project |
| Brief on target attendees | Notalone |

**During Conference:**
| Task | Owner |
|------|-------|
| House event execution | Joint |
| Speaking/panel support | Notalone |
| Introductions to targets | Notalone |
| Lead capture | Project |

**Post-Conference (within 2 weeks):**
| Task | Owner |
|------|-------|
| Lead follow-up | Project |
| Token release (if Activity Pool) | Notalone/Smart Contract |
| Debrief and learnings | Joint |
| Plan next activation | Joint |

### 7.2 Token Release Triggers

| Pool | Trigger | Verification |
|------|---------|--------------|
| Base Pool | Monthly time-based | Automatic |
| Activity Pool | Conference participation confirmed | Multisig approval |
| Performance Pool | Milestone achieved | Evidence review |

**Deliverable:** Activation report, token release executed

## Stage 8: Monitoring & Exit

### 8.1 Ongoing Monitoring

| Metric | Frequency | Source |
|--------|-----------|--------|
| Token price | Daily | Price feeds |
| On-chain metrics | Weekly | Dashboards |
| Team updates | Bi-weekly | Calls/async |
| Growth Assets utilization | Per event | Internal tracking |
| Milestone progress | Monthly | Project reports |

### 8.2 Portfolio Review

**Quarterly Review:**
- Token performance vs. thesis
- Growth Assets activation rate
- Milestone progress
- Relationship health
- Continuation/exit decision

### 8.3 Exit Triggers

| Trigger | Action |
|---------|--------|
| Target price reached | Begin systematic exit |
| Thesis invalidated | Review exit options |
| Major unlock approaching | Hedge or exit decision |
| Project pivot/shutdown | Exercise exit provisions |
| Fund term approaching | Orderly liquidation |

### 8.4 Exit Execution

| Exit Type | Method |
|-----------|--------|
| Market sale | VWAP over X days |
| OTC secondary | Sell to other buyers |
| Strategic sale | Sell to project/foundation |
| Hold to maturity | Fund term distribution |

**Deliverable:** Exit memo, execution report
