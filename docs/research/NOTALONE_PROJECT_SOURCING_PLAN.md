## Executive Summary

This document outlines a systematic approach for identifying post-TGE investment targets aligned with Notalone's 80/20 fund structure. The primary focus is on the **Liquid Alpha (80%)** strategy, which targets post-TGE OTC deals with 30-50% discounts and 6-12 month vesting.

**Key Insight:** Notalone is fundamentally a **liquid token trading operation** with a venture sleeve, not a traditional VC fund. The sourcing strategy must prioritize projects with:
1. Active trading and liquidity (enables hedging)
2. Treasury pressure (enables discounts)
3. Growth gaps (enables value-add positioning)

## Part 3: Target Project Identification Criteria

### 3.1 Quantitative Screening Criteria

#### Tier 1 Filters (Must Pass All)

```
1. Has live token (post-TGE)
2. Listed on at least 2 CEXs
3. Daily volume >= $500K
4. FDV <= $100M (exceptions for tier-1)
5. Circulating supply >= 10%
6. Not in excluded sectors (gaming, metaverse, NFT)
```

#### Tier 2 Scoring Criteria (Score 0-100)

| Metric | Weight | Score Criteria |
|--------|--------|----------------|
| **Treasury Pressure** | 25% | <6mo runway = 25, 6-12mo = 15, >12mo = 5 |
| **Token Performance** | 20% | -50%+ from ATH = 20, -30% = 15, flat = 10 |
| **Growth Gap** | 15% | Declining users = 15, flat = 10, growing = 5 |
| **Sector Alignment** | 15% | Priority sector = 15, neutral = 10, avoid = 0 |
| **Team Quality** | 10% | Known + experienced = 10, known = 7, anon = 0 |
| **Hedging Feasibility** | 10% | Tier-1 CEX perps = 10, Tier-2 = 5, DEX only = 0 |
| **Unlock Schedule** | 5% | Short cliff = 5, medium = 3, long = 1 |

**Investment Threshold:** Score >= 60 to proceed to DD

### 3.2 Qualitative Assessment Framework

#### Red Flags (Immediate Rejection)
- Anonymous team members
- No product/MVP after 6+ months
- Unrealistic token utility claims
- Clone without differentiation
- Regulatory red flags (unregistered securities)
- Team conflicts or governance issues

#### Green Flags (Priority Investigation)
- Team previously built successful projects
- Real on-chain usage (not just TVL from incentives)
- Strong ecosystem partnerships
- Clear path to revenue/sustainability
- Active development (GitHub commits)
- Community growth driven by product (not airdrops)

## Part 4: Screening Workflow

### 4.1 Weekly Screening Pipeline

```
Stage 1: Automated Scan (Daily)
├── Pull new TGEs from past 30-180 days
├── Filter by quantitative criteria
├── Score against Tier 2 metrics
└── Output: Candidate list (20-50 projects)

Stage 2: Rapid Assessment (Weekly)
├── Review candidate list
├── 5-10 minute qualitative check
├── Apply red/green flags
└── Output: Shortlist (5-10 projects)

Stage 3: Deep Dive (Weekly)
├── Full DD on shortlisted projects
├── Team background verification
├── Tokenomics analysis
├── Competitive positioning
└── Output: IC-ready projects (2-3)

Stage 4: Outreach (Ongoing)
├── Warm intro via network
├── Direct founder contact
├── OTC desk coordination
└── Output: Active negotiations
```

### 4.2 Screening Questions (Per Project)

**Core Questions (5 min):**
1. What does the project do? (1 sentence)
2. When was TGE? Current FDV?
3. Current ATH drawdown?
4. Daily volume? Which exchanges?
5. What sector? In priority list?

**DD Questions (30 min):**
1. Team: Who are they? Previous experience?
2. Product: Live product? Real users?
3. Treasury: What's their runway?
4. Unlock schedule: When do tokens unlock?
5. Why OTC: Why are they selling at discount?
6. Hedging: Can we hedge on perp markets?
7. Value-add: What growth support do they need?

## Part 5: Database Enhancement Plan

### 5.1 Current Database State

The existing PostgreSQL schema on XNode3 is designed for the **Israeli tech ecosystem** (LP sourcing), not investment targets. For post-TGE project tracking, we need to enhance or create new tables.

### 5.2 Proposed Schema Extensions

#### Option A: Extend Existing Schema

Add new fields to `notalone_companies` table:

    token_address VARCHAR(100),

    circulating_market_cap BIGINT,
    circulating_supply_percent DECIMAL(5,2),
    daily_volume_usd BIGINT,

    exchange_listings JSONB DEFAULT '[]',

#### Option B: Create New Investment Targets Table

```sql

    daily_volume_usd BIGINT,

    ath_price DECIMAL(20,8),
    ath_date DATE,
    current_price DECIMAL(20,8),
    ath_drawdown_percent DECIMAL(5,2),

    -- On-Chain Metrics
    tvl_usd BIGINT,
    monthly_active_users INTEGER,
    daily_active_users INTEGER,
    monthly_transactions INTEGER,
    monthly_revenue_usd BIGINT,

    -- Exchange & Liquidity
    exchange_listings JSONB DEFAULT '[]',
    has_perp_futures BOOLEAN DEFAULT FALSE,
    perp_exchanges JSONB DEFAULT '[]',
    liquidity_score INTEGER,  -- 1-100

    -- External IDs
    coingecko_id VARCHAR(100),
    coinmarketcap_id VARCHAR(100),
    defillama_id VARCHAR(100),
    rootdata_id VARCHAR(100),

    -- Team
    team_members JSONB DEFAULT '[]',
    team_is_known BOOLEAN DEFAULT TRUE,
    team_previous_projects JSONB DEFAULT '[]',

    -- Screening Score
    screening_score INTEGER,  -- 0-100
    tier1_passed BOOLEAN DEFAULT FALSE,
    tier2_score JSONB,  -- Individual component scores

    -- Pipeline Status
    pipeline_stage VARCHAR(50),
    deal_source VARCHAR(255),
    source_contact VARCHAR(255),
    assigned_analyst VARCHAR(100),
    first_contact_date DATE,
    last_contact_date DATE,
    next_action TEXT,
    next_action_date DATE,

    -- Deal Terms (if in negotiation)
    offered_discount_percent DECIMAL(5,2),
    vesting_months INTEGER,
    cliff_months INTEGER,
    tge_unlock_percent DECIMAL(5,2),
    check_size_usd BIGINT,

    -- Assessment
    investment_thesis TEXT,
    key_risks TEXT,
    value_add_opportunity TEXT,
    comparable_investments JSONB DEFAULT '[]',

    -- Decision
    decision VARCHAR(50),  -- Invested, Passed, On Hold
    decision_date DATE,
    decision_reason TEXT,

    -- Metadata
    notes TEXT,
    attachments JSONB DEFAULT '[]',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Indexes for common queries
CREATE INDEX idx_targets_pipeline ON notalone.notalone_investment_targets(pipeline_stage);
CREATE INDEX idx_targets_sector ON notalone.notalone_investment_targets(sector);
CREATE INDEX idx_targets_score ON notalone.notalone_investment_targets(screening_score);

CREATE INDEX idx_targets_volume ON notalone.notalone_investment_targets(daily_volume_usd);
CREATE INDEX idx_targets_next_action ON notalone.notalone_investment_targets(next_action_date);
```

### 5.3 Recommended Approach

**Use Option B** (separate table) because:
1. Current schema focuses on Israeli ecosystem for LP sourcing
2. Investment targets need different fields and update frequency
3. Separation of concerns: LP prospects vs investment targets
4. Can link via `company_id` FK when relevant

## Part 6: Automated Data Collection

### 6.1 Data Pipeline Architecture

```
Data Sources          ETL Pipeline           Database          Dashboards
─────────────         ────────────           ────────          ──────────
CoinGecko API    ──┐
DefiLlama API    ──┼──> Python Scripts ──> PostgreSQL ──> Superset
Token Unlocks    ──┤    (daily cron)       (XNode3)        Dashboards
Artemis API      ──┤
Crunchbase       ──┘
```

### 6.2 Recommended Python Scripts

**Script 1: `fetch_coingecko_data.py`**
- Fetch all tokens from CoinGecko API
- Filter by criteria (volume, market cap)
- Extract: price, volume, FDV, exchanges
- Update `notalone_investment_targets`

**Script 2: `fetch_defillama_data.py`**
- Fetch TVL data for protocols
- Match to tokens by address/name
- Extract: TVL, fees, revenue
- Update metrics fields

**Script 3: `fetch_token_unlocks.py`**
- Fetch upcoming unlock events
- Identify projects with treasury pressure
- Flag for outreach opportunity

**Script 4: `calculate_screening_scores.py`**
- Apply Tier 1 filters
- Calculate Tier 2 scores
- Update pipeline stages

### 6.3 API Rate Limits & Costs

| API | Rate Limit | Cost | Notes |
|-----|------------|------|-------|
| CoinGecko (free) | 10-30 calls/min | Free | Limited to 250 coins/page |
| CoinGecko (Pro) | 500 calls/min | $129/mo | Full access |
| DefiLlama | Unlimited | Free | Open source |
| Token Unlocks | 100/day (free) | Free | Basic data |
| Artemis | 60/min | $500+/mo | Premium data |

**Recommendation:** Start with free tiers (CoinGecko free, DefiLlama, Token Unlocks), upgrade as needed.
